# Intent Sharpening — 의도 깎기

기획자의 머릿속에 있는 흐릿한 의도를 **소크라테스식 문답**으로 깎아내는 Claude Code 플러그인.

## 작동 방식

1. 모호한 의도를 한 줄로 던진다 ("AI 추천 기능을 넣고 싶어")
2. 소크라테스식 질문으로 의도를 깎는다 (라운드당 최대 2개)
3. 별도 **Clarity Judge** 에이전트가 6축 기준으로 명료함을 판정한다 (0~100점)
4. 80점 이상이 될 때까지 반복한다 (기본 최대 20라운드, 설정 가능)
5. 최종 **Intent Card**를 생성한다

핵심 원리: **답을 주지 않는다. 질문으로 깎는다.**
돌덩이(모호한 의도) → 소크라테스식 문답 → 조각상(명료한 의도)

**중요: "의도"만 깎는다. "구현"은 깎지 않는다.**
- 깎는 것: Why(왜), What(무엇이 달라지는가), Who(누구를 위해), 범위, 성공기준, 리스크
- 깎지 않는 것: How(어떻게 만들 것인가) — 기술 스택, 아키텍처, UI 등

## 설치

```bash
npx skills add cosmos-makers/intent-sharpening
```

## 사용법

```
/intent-sharpening:intent-sharpening 우리 서비스에 AI 추천 기능을 넣고 싶어
```

인자 없이 실행하면 의도를 물어본다:

```
/intent-sharpening:intent-sharpening
```

최대 라운드 설정 — `--rounds` 또는 자연어로:

```
/intent-sharpening:intent-sharpening --rounds 5 대시보드 만들기
/intent-sharpening:intent-sharpening 대시보드 만들기, 3라운드만 하자
```

## 평가 기준

| 축 | 배점 | 기준 |
|----|------|------|
| 대상 (Who) | 15 | 누구를 위한 것인지 특정됨 |
| 동기 (Why) | 20 | 왜 해야 하는지 근거가 있음 |
| 결과물 (What) | 20 | 구체적 산출물/변화가 보임 |
| 범위 (Scope) | 15 | 포함/제외가 구분됨 |
| 성공기준 (Measure) | 15 | 검증 가능한 기준이 있음 |
| 리스크 (Risk) | 15 | 주요 리스크나 제약을 인지함 |

**합계 100점. 80점 이상이면 실행 가능한 수준.**

## Intent Card

**한 줄 요약**: 콘텐츠 상세 페이지 이탈률을 개인화된 관련 콘텐츠 추천으로 낮춘다

**대상(Who)**: 검색/소셜에서 콘텐츠 상세 페이지로 랜딩하는 사용자
**동기(Why)**: 콘텐츠 페이지 이탈률 68% — 탐색 없이 이탈
**결과물(What)**: 콘텐츠 상세 페이지 하단 "관련 콘텐츠" 섹션, 시청 이력 기반 개인화
**범위(Scope)**: 콘텐츠 상세 페이지만. 피드, 검색 결과, 홈은 제외
**성공기준(Measure)**: 이탈률 68% → 55% 이하 (4주 내)
**리스크(Risk)**: 신규 사용자 콜드스타트 → 인기순 폴백

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

🪨 의도 깎기 시작

원문: "에이전트를 사용하는 사용자들을 위해 팀 컨텍스트를 마운트 언마운트하는 도구를 제공하고 싶어."

Round 1 — 첫 번째 깎기
이걸 왜 해야 해? 지금 팀 컨텍스트 마운트/언마운트 없이 뭐가 문제야?
"팀 컨텍스트"가 뭔지 좀 더 풀어줘. 사용자 입장에서 이게 없으면 구체적으로 뭘 못 하는 거야?

> 1. 팀 컨텍스트는 팀 공통의 메모리야. 작업 담당자, 스펙, 용어사전, 코드레포 위치,
>    위키 위치 같은 신입사원 팀 적응에 필요한 모든 맥락이 담긴 마크다운 폴더.
> 2. 누구든 해당 팀 컨텍스트를 마운트하면 쉽게 팀 지식을 에이전트에 접목할 수 있어.
> 3. 이게 없으면 매번 헤비하게 정보를 찾고 탐색해야지.

📊 Round 1 명료함 판정: 52/100
▓▓▓▓▓▓▓▓▓▓░░░░░  Who     10/15  에이전트 사용자 — 신입? 숙련자? 더 좁혀지지 않음
▓▓▓▓▓▓▓▓▓▓▓▓▓▓░░░░░░  Why     14/20  정보 탐색 비용 절감 — 정량적 근거 부족
▓▓▓▓▓▓▓▓▓▓▓▓░░░░░░░░  What    12/20  마운트/언마운트 도구 — 마운트하면 뭐가 달라지는지 불명확
▓▓▓▓▓▓▓▓░░░░░░░  Scope    8/15  구성요소는 언급 — 경계 미정

▓▓▓░░░░░░░░░░░░  Risk     4/15  리스크 인식 거의 없음
가장 약한 축: Measure, Risk

Round 2 — 계속 깎는다
이게 성공하면 어떻게 알아? "팀 컨텍스트 마운트한 에이전트가 이걸 할 수 있으면 된다" 같은 게 뭐가 있어?
이 컨텍스트가 오래되면 어떻게 돼? 최신화 안 된 컨텍스트를 마운트한 에이전트가 틀린 정보로 일하면?

> 성공하면 사용자가 에이전트와 대화 중에 팀 관련 개념을 물어보면 에이전트가
> 팀 컨텍스트에서 찾아 답변하거나, 굳이 안 물어도 작업할 때 팀 지식을 활용해.
>
> 컨텍스트가 오래되면 동기화 문제가 생기지. 마운트 시 레포의 최신 커밋 헤드와
> 비교하면 될 것 같아. (레포식 관리가 맞다고 봐 — 여러 명이 공유해야 하니까)

## Intent Card

**한 줄 요약**: 에이전트 사용자가 팀 공통 지식(팀 컨텍스트)을 마운트/언마운트하여
에이전트가 팀 스페시픽한 업무를 즉시 수행할 수 있게 하는 도구

**대상(Who)**: 에이전트를 사용하는 조직 구성원 (전사/팀/스쿼드 단위)

**동기(Why)**: 팀 컨텍스트 없이는 매번 헤비하게 정보 탐색. 마운트하면 에이전트가
팀 지식(담당자, 스펙, 용어사전, 코드레포, 위키 위치)을 바로 활용

**결과물(What)**:
- 마운트/언마운트/리스트 확인 기능
- 팀 컨텍스트 편집 → 공용 레포 푸시
- 마운트 시 레포 최신 커밋 헤드 비교로 동기화 확인
- 에이전트가 명시적 질문 없이도 팀 지식 기반으로 업무 수행

**범위(Scope)**:
- 공유 스코프: 전사 / 팀 / 스쿼드
- 여러 팀 컨텍스트 동시 마운트 가능 (보통은 하나)
- 편집+푸시 포함, 레포식 관리
- 권한은 레포 권한 따름, PR 거칠지 여부는 레포별 설정

**성공기준(Measure)**: 각 팀이 팀 컨텍스트를 만들고, 사용자가 자유롭게
마운트·확인·언마운트하며 에이전트에게 풍성한 맥락을 전달하는 상태

**리스크(Risk)**:
- 동기화 — 레포 커밋 헤드 비교로 대응
- 잘못된 편집 전파 — PR/롤백(레포 버전관리)
- 권한 — 레포 접근 권한 기반 관리

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

⚠️ 아직 약한 축:
- Measure: 성공 이미지는 있지만 정량 지표 없음
- Risk: 보안 민감 정보, 대형 컨텍스트 성능, 온보딩 장벽 미다룸
```
